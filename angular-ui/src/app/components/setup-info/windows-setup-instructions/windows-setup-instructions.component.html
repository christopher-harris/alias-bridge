Activate Aliases on Windows (Git Bash / WSL)

On Windows, AliasBridge currently saves aliases in a format compatible with Bash shells like Git Bash or within the Windows Subsystem for Linux (WSL). This method does not directly add aliases to standard PowerShell or Command Prompt (cmd.exe).

Instructions:

  1. If using Git Bash:

    Add the following lines (preferably at the end) to your user's .bashrc or .bash_profile file (usually located at C:\Users\YourUsername\.bashrc):

    # Load AliasBridge Aliases
if [ -f ~/.alias_bridge_aliases.sh ]; then
  source ~/.alias_bridge_aliases.sh
fi

    (Git Bash usually understands ~ correctly to refer to your Windows home directory where AliasBridge saves the file.)

  2. If using WSL (e.g., Ubuntu):

    Add the following lines to your .bashrc or .zshrc file within your WSL distribution (e.g., inside Ubuntu, run nano ~/.bashrc):

    # Load AliasBridge Aliases from Windows Home Directory
# Replace 'YourWindowsUsername' with your actual Windows username
WINDOWS_ALIAS_FILE="/mnt/c/Users/YourWindowsUsername/.alias_bridge_aliases.sh"
if [ -f "$WINDOWS_ALIAS_FILE" ]; then
  source "$WINDOWS_ALIAS_FILE"
fi

    You need to replace YourWindowsUsername with your actual Windows user folder name. The path starts with /mnt/c/ to access your C: drive from within WSL.

After adding the lines to your chosen configuration file, you must open a new Git Bash or WSL terminal session for the changes to take effect.

Note: Native PowerShell support might be added in the future.
